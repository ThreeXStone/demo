# 更新踩坑记录

## 立即执行

检查项目根目录的 `CLAUDE.md`：

1. 如果不存在 `## 踩过的坑` 章节，创建它
2. 将本次对话中发现的问题和解决方法追加进去
3. 每条记录格式如下：

```
- ❌ 错误做法：`<具体命令或操作>`
  ✅ 正确做法：`<具体命令或操作>`
  📌 原因：<一句话说明为什么>
```

## 本次需要记录的内容

- ❌ 错误做法：`npm install` 或 `npm add`
  ✅ 正确做法：`bun add <package> --cwd <子项目路径>`
  📌 原因：本项目使用 Bun 作为包管理器，根目录 package.json 已指定 `packageManager: bun`，且各子项目需通过 `--cwd` 指定路径

## 长期规范

在后续所有任务中，每当遇到报错、走弯路、或纠正了错误做法后，主动将经验追加到 `CLAUDE.md` 的 `## 踩过的坑` 章节，无需等待用户提醒。

判断标准：
- 用错了命令 → 记录
- 配置文件写错格式 → 记录  
- 端口/路径/服务名搞错 → 记录
- 用户纠正了你的做法 → 记录
